# 🌱 Growth path — co dodawać, gdy system rośnie (i po czym poznać, że CZAS)

> Seed celowo startuje chudy. Każdą warstwę dodawaj **dopiero, gdy poczujesz jej sygnał** — nie na zapas.
> Budowanie na zapas = najczęstszy sposób porzucenia second braina: system „gotowy", a używać nie ma komu.

| Etap | Co dodajesz | Sygnał, że czas (realne tarcie) |
|---|---|---|
| 0 | **Core loop:** `CLAUDE.md` + `INDEX.md` + `INBOX.md` + git | day 1 — jest w seedzie |
| 1 | **📍 Handoff + rytuał CLOSE** („nadpisz Handoff i zacommituj") | agent na starcie sesji pyta „od czego zaczynamy?", a Ty tłumaczysz kontekst od zera |
| 2 | **Rytuał START** (agent sam czyta Handoff+NOW i proponuje plan) | zapominasz kazać mu przeczytać stan; sesje zaczynają się chaotycznie |
| 3 | **`raw/`** (surowiec append-only) | wartościowa synteza / research WYPAROWAŁA z czatu i musisz ją odtwarzać (tokeny kosztują 2×) |
| 4 | **Scout na serio** (cykliczny przegląd soczewkami + [`cadence.md`](cadence.md)) | pierwszy realny desync: doc kłamie, link wisi, todo zrobione-a-nieodhaczone |
| 5 | **[`decisions.md`](decisions.md) na serio** | drugi raz rozważasz pomysł, który już kiedyś odrzuciłeś — i nie pamiętasz czemu |
| 6 | **Skille / komendy** (powtarzalne procedury agenta) | to samo tarcie **≥2–3×** i za każdym razem tłumaczysz agentowi to samo |
| 7 | **Hooki / automatyczny start sesji** | rytuał ręczny działa, ale nudzi i zaczynasz go pomijać |

## Anty-wzorce
- ❌ Etapy 4–7 w tygodniu pierwszym — zbudujesz system zamiast go używać.
- ❌ Automatyzacja czegoś, czego ręcznie nie zrobiłeś ani razu.
- ❌ Warstwa „bo może się przyda" — każda dokłada stały koszt utrzymania i kontekstu; bez konsumenta = balast.
- ❌ Optymalizowanie czegoś, co powinno zostać skasowane (kolejność: usuń → uprość → dopiero automatyzuj).
